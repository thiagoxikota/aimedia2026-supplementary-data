# Product Brief: Fleet Management Dashboard

## Context

This brief was provided identically to both conditions (AI tool and human designer) as the sole input for prototype generation. No additional instructions, references, or iterations were given beyond this document.

## Product Description

Design a web-based fleet management dashboard for a logistics company operating in urban Brazil. The application serves operations managers who monitor drivers, deliveries, and fleet performance in real time.

## Functional Requirements

The application must include the following functional areas:

1. **Dashboard** — Overview screen with key performance indicators (active drivers, pending deliveries, completed deliveries, alerts), summary charts, and quick-access navigation to other sections.

2. **Motoristas (Drivers)** — Driver listing with status (available, en route, offline), current route assignment, vehicle information, and delivery count. Support for search and filtering.

3. **Entregas (Deliveries)** — Delivery tracking with status (pending, in transit, delivered, failed), assigned driver, destination address, and estimated time of arrival.

4. **Alertas (Alerts)** — Operational alert center showing severity levels (critical, warning, info), alert source (system, driver, vehicle), timestamps, and resolution status.

5. **Mapa (Map)** — Geographic view showing real-time driver positions and active delivery routes on a map interface.

## Design Constraints

- Desktop-first layout (minimum viewport: 1366x768)
- Professional, data-dense interface appropriate for operations centers
- Portuguese (Brazilian) language for all labels and content
- Dark or light theme at designer's discretion

## Deliverable

A navigable high-fidelity prototype with realistic placeholder data, covering the functional areas above. Each functional area should have at least one screen. Navigation between sections must be functional.

## Notes

- No brand guidelines, color palette, or typography were specified — both conditions were free to make visual design decisions
- No component library or design system was prescribed
- The brief was delivered as a single text prompt to the AI tool and as a written document to the human designer
