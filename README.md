# Supplementary Visual Dataset

**Structural Convergence in AI-Generated Media: How Specification Literacy Moderates Homogenization in GenAI-Assisted Interface Design**

AIMEDIA 2026

---

## Overview

This repository contains the complete visual dataset for our study comparing UI screens generated by an AI design tool (Figma Make) against screens designed by a human designer, both working from an identical product brief.

Each screen represents a distinct page within a navigable prototype. The numbered pages correspond to different views encountered during prototype navigation (e.g., overview, listing, detail, filtered states). Full-resolution exports were captured from Figma via the REST API at 2x scale in PNG format.

A downloadable PDF compiling the full dataset is available: [`supplementary_visual_dataset.pdf`](supplementary_visual_dataset.pdf). The PDF can be regenerated from source images: `pip install -r requirements.txt && python generate_pdf.py`.

---

## Conditions and Scope Asymmetry

Both conditions received an identical product brief specifying five functional areas: Dashboard, Drivers, Deliveries, Alerts, and Map. No component library, color palette, or design system was prescribed.

The AI condition (Figma Make) produced 21 screens across 7 functional areas. The Designer condition produced 9 screens across 5 functional areas. This 2.3:1 asymmetry is an expected outcome of the study design, not a methodological limitation:

- The AI tool generates multiple layout variations per prompt, producing broader but shallower coverage of each functional area.
- The human designer made deliberate navigation architecture decisions, producing fewer screens with deeper information hierarchy (e.g., dedicated driver detail views absent from the AI output).
- Both conditions worked from the same brief with no time or scope constraints. The designer self-determined when the prototype was complete; the AI output was accepted as-is from a single generation pass.

This difference in volume and coverage is itself a data point discussed in the main paper (Section 4.2).

## Methodology

### Ethics and Consent

All participants provided informed consent via the study's online form (TCLE declaration) prior to data collection. Participation was voluntary and uncompensated. No personally identifiable information is present in the dataset. Ethics review details are reported in the main paper.

### AI Condition

Screens were generated using Figma Make (Figma's native generative design tool). The AI condition received the product brief as a single prompt with no iterative refinement — output was accepted as-is from one generation pass. Tool version and generation date are reported in the main paper (Section 3).

### Designer Condition

The human designer worked independently from the same product brief with no time constraints. The designer self-determined when the prototype was complete. Designer experience level and background are reported in the main paper (Section 3).

### Data Collection

All frames were exported from Figma via the REST API at 2x scale in PNG format. Viewport-cropped versions (1426px height) were generated programmatically by applying a fixed crop window from the top of each frame; no content was selectively removed or reordered.

### Image Descriptions

Image captions describe observable UI elements (navigation type, component presence, layout structure) rather than interpretive assessments. Descriptions were written by a single author after export, with knowledge of condition assignment. They are provided for navigational convenience and are not part of the analytical dataset. Caption categories (navigation type, component presence, layout structure) were applied consistently across both conditions.

## Survey Data

Anonymized participant data from the specification literacy survey (n=6). Participants were professional designers who received the same product brief and were asked to write a prompt for Figma Make.

- [`survey_responses.csv`](survey_responses.csv) — Quantitative response data (experience, familiarity scales, self-reported difficulty/confidence/pattern avoidance, prompt strategy classification)
- [`survey_prompts.md`](survey_prompts.md) — Full verbatim prompt texts with stated rationale
- [`survey_analysis.md`](survey_analysis.md) — Descriptive analysis of specification literacy patterns and prompt strategies

### Prompt Strategy Summary

| ID | Exp. | AI fam. | Strategy | Words | Confidence | Pattern avoidance |
|----|------|---------|----------|-------|------------|-------------------|
| P1 | 7 yr | 4 | Minimal extraction | ~120 | 4 | 3 |
| P2 | 4 yr | 3 | Contextual specification | ~350 | 3 | 4 |
| P3 | 3.2 yr | 5 | Exhaustive specification | ~2500 | 4 | 5 |
| P4\* | 7 yr | 3 | Meta-description | ~40 | 4 | 1 |
| P5 | 4 yr | 1 | AI-delegated | ~200 | 1 | 1 |
| P6 | 3.5 yr | 3 | Structural specification | ~500 | 4 | 3 |

\* P4 described their approach but did not submit a prompt. Excluded from convergence analysis in the main paper (n=5).

See [`survey_analysis.md`](survey_analysis.md) for detailed pattern analysis supporting the paper's specification literacy framework.

### Ordering Convention

Files are numbered sequentially (fig_01–fig_30) grouped by functional area, with the AI condition listed before the Designer condition within each group. This ordering is editorial and does not reflect chronological generation order. The `author` field in `manifest.json` is the authoritative condition label.
